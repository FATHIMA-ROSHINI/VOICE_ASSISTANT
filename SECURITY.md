| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security issue, please report it responsibly.

### How to Report

1. **Do NOT** create a public GitHub issue for security vulnerabilities
2. **Email** us directly at: rxzrosh@gmail.com
3. **Include** in your report:
   - Description of the vulnerability
   - Steps to reproduce the issue
   - Potential impact assessment
   - Any suggested fixes (optional)

### What to Expect

- **Acknowledgment**: We aim to acknowledge your report within 48 hours
- **Update**: We'll provide regular updates on the progress of addressing the vulnerability
- **Credit**: We will credit you in the security advisory (if desired)
- **Disclosure**: We request that you give us reasonable time to address the issue before public disclosure

### Scope

This security policy applies to:
- This repository
- All associated branches
- Related services and deployments

### Out of Scope

- Social engineering attacks
- Physical security
- Denial of service attacks (unless directly related to a code vulnerability)

## Security Best Practices

When using this project, we recommend:

- Keep dependencies updated
- Use environment variables for sensitive data
- Follow principle of least privilege
- Implement proper authentication and authorization
- Regular security audits

This project follows industry security standards including:
